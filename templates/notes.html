Development Notes

I've been playing around with a few ideas for the Lojban website, and I set this up temporarily to get a bit of feedback. It's only a rough draft at the moment, so I haven't done any browser testing or compatibility work. I'm developing with Firefox 2 at 1280x1024, but you shouldn't have any serious problems with any modern browser with default settings. It's likely to fall apart in Internet Explorer unless you are using a version 8 beta. For people having difficulty, take a look at the screenshot.

Goals for the website

  • Public face of the Lojban project.
  • Audience is anybody new or semi-new to Lojban.
  • Shows people what Lojban is like.
  • Teaches people how to speak it.
  • Acts as a hub for the community.

Problems with the existing site

  • Overcomplicated and crowded (150 links on the homepage alone).
  • Material suitable for newcomers is mixed up with material only of interest to experts with no clear distinction between the two.
  • Lots of low quality material reducing the signal:noise ratio (outdated, irrelevant information, broken links, etc).
  • Incohesive; mailing list archives are dissociated from the rest of the site, same goes for the books, main vocabulary links are to unformatted text/data files.

Get involved

Patches, suggestions and criticism are all welcome. I've set up a temporary mailing list that you can subscribe to that is intended solely for the website redesign. It's probably a good idea to catch up with the archives and the links to the right before posting to the list.

Please report any bugs, wishlist items, etc, by opening a new ticket on Trac.

So far, the Django framework and the git revision control system have been chosen. Right now, the pages are mostly generated from static templates. You can pull the code from github.com with: 

git clone git://github.com/JimDabell/lojban-website.git

From there, look at docs/contributing.txt.
